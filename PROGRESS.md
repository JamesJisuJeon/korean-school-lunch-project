## 🛠 구현된 주요 기능 상세

### 1. 인증 및 보안 (Authentication)
- **Role 기반 접근 제어:** ADMIN, PA, TEACHER, PARENT 권한별 메뉴 노출 및 API 보안 적용.
- **계정 관리:** 이메일 아이디 사용, 최초 로그인 시 비밀번호 변경 강제, 관리자용 비밀번호 초기화 기능.

### 2. 학교 조직 및 학생 관리 (School Structure)
- **학년도 관리:** 연도별 학년도(Academic Year) 생성 및 활성화 설정.
- **학급 관리:** 각 학년도에 종속된 반(Class) 생성 및 선생님 배정.
- **학생 관리:** 학부모 계정과 연동된 자녀 등록, 다자녀 지원, 학부모회(PA) 자녀 무료 혜택 자동 판별.

### 3. 고도화된 주간 메뉴 시스템 (Advanced Menu System)
- **메뉴 세분화:** 메인 메뉴, 디저트, 음료수 3가지 카테고리별 등록 기능.
- **이미지 업로드:** 안내 포스터 등 이미지를 서버에 직접 업로드하고 미리보기 제공.
- **NZ 시간 기반 자동 마감:**
  - 배식 날짜(토요일) 선택 시 해당 주 목요일 낮 12시(NZ 시간) 자동 마감 설정.
  - 마감 시간 이후 학부모의 온라인 신청 자동 차단.
- **게시 제어:** 학부모회에서 게시 확정한 메뉴만 학부모 신청 화면에 노출.

### 4. 현장 운영 및 실시간 연동 (Operations)
- **학부모회 현장 관리:** 엑셀을 대체하는 실시간 수납 확인, 현장 즉석 신청, 매점 쿠폰($5) 판매 기록.
- **선생님용 대시보드:** 담당 학급 학생들의 점심 신청 및 수납 여부 실시간 조회 (배식 확인용).
- **학부모 신청:** 자녀별 맞춤 가격($0 또는 메뉴가) 자동 계산 및 신청 내역 확인.

### 5. UI/UX 및 가독성 개선
- **선명한 디자인:** 모든 입력창(Input, Select)의 글씨체와 테두리를 진하게 수정하여 가독성 확보.
- **반응형 대시보드:** 모바일 및 데스크탑에서 최적화된 카드형 레이아웃 적용.
- **다크 모드 지원:** `next-themes`를 활용한 라이트/다크/시스템 테마 선택 기능 추가 (`ThemeSelector` 컴포넌트).

### 6. 시스템 관리 및 UI 패턴 통일화 (2026-04)
- **UI/UX 통합:** 모든 관리자 모듈(사용자, 학급, 학생, 보결선생님)의 디자인을 헤더(액션바) + 목록(테이블) 형태로 표준화 완료.
- **시스템 관리 모드 토글:** 관리자 권한 보유 시, 실수 방지를 위해 '시스템 관리 모드'를 명시적으로 켜고 끌 수 있는 토글 기능 추가 (`AdminToggle` 컴포넌트).
- **권한 및 메뉴 노출 최적화:**
  - 보결 선생님 등록 시 당일 한정으로 선생님 메뉴 노출 허용.
  - 관리자 역할을 가진 시스템 관리자일지라도 담임 학급이 배정되어 있거나 보결선생님인 경우에만 학급 관리 메뉴가 보이도록 최적화.
- **데이터 무결성 결함 수정:**
  - UI에서 담임교사를 텍스트 모드로 입력 후 저장 시 이전 데이터가 남는 상태 동기화 버그 픽스.
  - 엑셀 업로드 시 이름 기반 ID 매칭 실패 시 잘못된 ID가 유지되는 결함 수정.
  - ID 값이 부재(`null`)인 경우에는 표면적인 텍스트(`teacherName`)만 저장되는 것을 서버단(API)에서 원천 차단하여 무결성 확보.

### 7. 점심 신청 시스템 고도화 (2026-04)
- **자녀 다중 선택 및 일괄 신청:** 학부모가 여러 자녀를 체크박스로 선택하여 한 번에 점심을 신청할 수 있도록 UX 개선.
- **특이사항(알러지 등) 기록:** 자녀별로 주문 시마다 개별 특이사항을 입력할 수 있는 기능 추가. (주문 내역에서 수정 가능)
- **메뉴 노출 최적화:** 게시된 메뉴가 하나인 경우 불필요한 선택 과정 없이 즉시 메뉴 정보가 보여지도록 화면 구성 단순화.
- **신청 상태 관리(취소/변경):** 신청 마감 전이며 아직 수납 전인 경우, 학부모가 직접 신청 내역에서 취소할 수 있는 기능 추가.
- **데이터 엔드포인트 무결성:** 누락되었던 자녀 조회 API(`/api/parent/students`)를 신규 생성하여 자녀 선택이 되지 않던 버그 해결.
- **Batch API 지원:** 여러 명의 학생을 동시에 처리할 수 있도록 주문 API(POST) 고도화.
- **PA 무료 대상자 정책 보완:** 학부모회(PA) 자녀의 $0 신청 건도 즉시 '수납완료' 되지 않고 '수납대기'로 시작하도록 하여, 신청 마감 전까지 취소가 가능하도록 UX 개선.

### 8. 현장 수납 및 매점 관리 고도화 (2026-04-04)
- **컬럼 분리 및 정렬:** 기존 통합 컬럼(학급/이름)을 학급·이름으로 분리하고 각 컬럼에 오름/내림차순 정렬 기능 추가 (한국어 로케일 정렬).
- **필터 기능 추가:** 학급, 사전/현장 신청 유형, 수납 상태별 드롭다운 필터링. 활성 필터 수 뱃지 및 초기화 버튼 제공.
- **현장 신청 버그 수정:** `orderType: "ONSITE"` → `"ON_SITE"` Prisma enum 오타 수정.
- **현장 신청 취소:** 수납 상태 드롭다운에서 '취소' 선택 시 ON_SITE 주문만 레코드 삭제(미신청 상태 복귀). 사전 신청(PRE_ORDER)은 상태만 변경.
- **현장 신청 기본 수납 상태 변경:** 현장 신청 생성 시 기본값을 `PAID` → `WAITING`(수납 대기)으로 변경.
- **수납 상태 / 특이사항 컬럼 분리:** 기존 통합 컬럼을 별도 컬럼으로 분리, 특이사항 입력 필드 글씨 크기 개선.
- **쿠폰비 수납 컬럼 추가:** `CouponSale` 모델에 `paymentStatus` 필드 추가(DB 마이그레이션), 매점 쿠폰 오른쪽에 납부/후납/후납-납부 드롭다운 표시.

### 9. UI 전면 개선 — 라이트/다크모드 완전 지원 (2026-04-04)
- **Tailwind v4 다크모드 설정:** `globals.css`에 `@custom-variant dark (&:where(.dark, .dark *))` 추가. 기존 CSS 클래스 오버라이드 방식(`!important`) 전면 제거.
- **전체 페이지 다크모드 적용:** 로그인, 비밀번호 변경, 대시보드, 메뉴 관리, 사용자 관리, 학년도 관리, 학급 관리, 학생 관리, 학급배정, 보결교사, 선생님 학급, 점심 신청, 현장수납 등 모든 화면에 `dark:` 클래스 일괄 적용.
- **점심 신청 화면 재설계:**
  - 레이아웃: 상단 바(이번주 메뉴 안내 + 날짜 + 가격 + 마감시간) + 중단 2컬럼(좌: 이미지, 우: 메뉴 박스 + 자녀 선택) + 하단 바(총금액 + 신청 버튼)
  - 메뉴 항목을 카테고리별 박스(Main/Dessert/Beverage) 형태로 표시.
  - 자녀 선택 시 특이사항 입력창 인라인 표시(1줄 높이, 스크롤바 숨김).
  - 메뉴 이미지 클릭 시 전체화면 확대 모달 기능 추가.
  - 나의 신청 내역을 신청하기 카드 하단에 배치, 취소 버튼 항상 표시.

### 10. 학급 관리(Teacher) 한글화 및 수납 상태 매핑 (2026-04-05)
- **컬럼 헤더 한글화:** NO→번호, STUDENT NAME→이름, SPECIAL NOTES→특이사항, STATUS→신청상태, PAYMENT→수납상태 (모두 가운데 정렬 통일).
- **요약 배지 한글화:** TOTAL→전체, ORDERED→신청, PAID→수납완료.
- **신청상태:** 신청완료 / 미신청 / 신청취소(CANCELLED일 때). 신청취소 시 신청 카운트에서 제외.
- **수납상태:** PAID·POST_PAID→수납완료(녹), UNPAID→후납(파), CANCELLED→취소(빨), WAITING→수납대기(노). animate-pulse 제거.

### 11. 운영 집계 현황 신규 기능 (2026-04-05)
- **신규 페이지:** `/pa/analytics` — 학부모회 운영 대시보드에 "운영 집계 현황" 메뉴 추가.
- **신청/수납 현황 (2열 구성):**
  - 1열(신청): 총인원 · 사전신청 일반(취소포함) · 사전신청 학부모회자녀(취소포함) · 총사전신청 · 사전신청취소 · 현장신청 · 최종신청인원
  - 2열(수납): 수납대기 · 납부/후납-납부 · 후납 · 최종확정인원(납부+후납 합계)
- **금액 현황 (7개 카드):** 점심납부총액 · 사전신청납부 · 사전신청후납 · 현장신청납부 · 현장신청후납 · 쿠폰판매금액 · 쿠폰판매후납.
- **반별 통계 테이블:** 반·총인원·신청·수납완료·후납·수납대기·취소·PA자녀·납부금액·쿠폰금액 + 합계 행.
- **API:** `/api/pa/analytics` 신규 생성 (menuId 기준 집계, 반별 sortOrder 정렬).

- **학급 신규 등록 폼:** 순서 입력 필드 추가 (숫자 전용, 스피너 없음).
- **테이블:** 순서 컬럼 표시 및 인라인 수정 지원. 컬럼 헤더 클릭으로 오름/내림차순 정렬 (학사연도·순서·학년·학급이름·담임교사).
